# Output

For each analyzed video, **2 files** are produced in the `output/` folder, plus optional screenshots.

---

## File 1: `_descriptions.txt`

**What it is:** chronological visual transcript of the video, frame by frame.

**Format:**
```
Frame 1 (t=0.0s): Initial application screen with navigation menu visible on the left. The user has not yet performed any actions.

Frame 2 (t=1.0s): The user clicks "New Order" in the side menu. A form opens with empty fields.

Frame 3 (t=2.0s): The customer name is typed in the first field. The field shows "John Smith".

Frame 4 (t=3.0s): The cursor moves to the "Product" field. A dropdown appears with the list of available products.
...
```

**Purpose:**
- Quick verification of what Claude saw at each moment
- Data source for the process analysis
- Useful for debugging if the final analysis seems inaccurate

---

## File 2: `_analysis.md`

**What it is:** structured process report, generated by Claude interpreting the descriptions.

**Full format:**

```markdown
# Video Analysis: tutorial.mp4

## 1. PROCESS OBJECTIVE
The user is creating a new order in the company management system,
entering customer data, selecting products, and confirming the order.

## 2. STEP-BY-STEP OPERATIONAL FLOW

1. **Accessing the orders section**
   - Action: click on "New Order" in the side menu
   - Interface: left navigation panel
   - Result: order entry form opens

2. **Entering customer data**
   - Action: typing name and personal details
   - Interface: form text fields
   - Result: customer "John Smith" selected

3. **Selecting products**
   - Action: opening dropdown and selecting product
   - Interface: dropdown with product list
   - Result: product added to order line

4. **Confirming quantity**
   - Action: modifying quantity field
   - Interface: numeric field
   - Result: quantity set to 3 units

5. **Saving the order**
   - Action: click on "Confirm Order"
   - Interface: primary button at the bottom of the form
   - Result: order saved, ID #10847 assigned

## 3. IDENTIFIED TECHNICAL ELEMENTS
- **System:** management ERP (web interface)
- **Data involved:** customer registry, product catalog, inventory
- **Visible technologies:** web application, standard HTML forms

## 4. CRITICAL OBSERVATIONS
- The process requires 5 distinct steps for a simple order
- No "quick order" function for returning customers
- Product dropdown becomes slow with large catalogs
- No real-time field validation (errors only discovered at save)

## 5. OPTIMIZATION SUGGESTIONS
- Add a customer search field with autocomplete
- Implement "recent orders" to reuse previous configurations
- Replace the product dropdown with full-text search
- Add live validation on required fields
- Consider a 2-step "quick order" flow for existing customers and products
```

## File 3: `output/frames/` *(optional)*

Available only with `--keep-frames`. Contains the extracted PNG screenshots:

```
output/
└── frames/
    └── tutorial/
        ├── frame_0001.png   ← t=0s
        ├── frame_0002.png   ← t=1s
        ├── frame_0003.png   ← t=2s
        └── ...
```

Each file is named `frame_NNNN.png` where NNNN is the sequential number (corresponding to the timestamp in the descriptions).

---

## Summary

| File | Location | Always present |
|---|---|---|
| `<name>_descriptions.txt` | `output/` | Yes |
| `<name>_analysis.md` | `output/` | Yes |
| `frames/<name>/frame_NNNN.png` | `output/frames/` | Only with `--keep-frames` |
